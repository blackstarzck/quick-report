# PRD: 업무 보고 사용자 화면 (Mobile-first MVP)

---

## 1. 개요

본 문서는 **업무 보고 서비스의 사용자 화면(User-facing)**을 대상으로 한  
**모바일 중심 MVP(Product Minimum Viable Product)** 기획 문서이다.

본 단계의 목적은 다음과 같다.

- 보고 작성 피로도 감소
- 장문 보고의 핵심 파악 용이
- 1일 보고 작성 후 즉시 공유 가능한 사용자 경험 제공

> 본 PRD는 **UX/UI 및 사용자 흐름 검증**을 우선하며,  
> 관리자 기능, 실제 AI API 연동, 카카오 SDK 연동은 범위에서 제외한다.

---

## 2. 대상 사용자

- 일일 업무 보고를 작성해야 하는 실무자
- 주 사용 환경이 **모바일 브라우저(PWA 포함)** 인 사용자

---

## 3. 문제 정의

1. 텍스트 나열식 보고로 인해 핵심 파악이 어렵다.
2. 상위 보고 작성을 위해 동일 내용을 반복 정리해야 한다.
3. 보고 빈도가 높아 작성 피로도가 누적된다.

---

## 4. 해결 방향

- 보고 내용에서 **핵심 키워드만 자동 추출**
- 일일 보고를 기반으로 **상위 보고 자동 집계가 가능한 구조 설계**
- 이전 보고 및 템플릿을 활용해 **반복 입력 최소화**
- 1일 보고는 **작성 직후 공유 가능한 UX 제공** (Mock 단계)

---

## 5. 기능 요구사항 (Functional Requirements)

### FR-1. AI 키워드 자동 추출

- 보고 제출 시 본문을 분석하여 **핵심 키워드를 자동 추출**한다.
- 키워드 수는 제한된 개수(예: 5~10개)로 제한한다.
- 키워드는 다음 위치에 노출된다.
  - 1일 보고 제출 완료 화면
  - 공유 미리보기 화면
- 키워드는 읽기 전 **빠른 스캔용 보조 정보**로 활용된다.

---

### FR-2. 보고 자동 집계 구조

- 보고는 다음 구조로 연결된다.

```
일일 → 주간 → 월간 → 결산
```

- 상위 보고 생성 시, 하위 보고를 기반으로 **초안을 자동 구성**한다.
- 자동 생성된 초안은 사용자가 편집 가능해야 한다.

---

### FR-3. 보고 재사용 및 템플릿

- 사용자는 보고 작성 시 다음 기능을 사용할 수 있다.
  - 전일/전주 보고 불러오기
  - 개인 업무 템플릿 삽입
- 불러온 내용은 **새 보고의 초안**으로 적용된다.
- 템플릿은 개인 단위로 관리한다.

---

### FR-4. 1일 보고 공유 (Mock)

- 1일 보고 제출 후 공유 전용 요약 화면을 제공한다.
- `카카오톡으로 공유` 버튼 UI를 제공한다.
- 실제 공유는 수행하지 않으며,
  `추후 연동 예정` 안내 UI를 노출한다.

---

## 6. User Flow (모바일 중심 + 카카오톡 공유 포함)

```
[앱 실행]
  ↓
[홈 – 오늘의 보고]
  ↓
[1일 보고 작성]
  ↓
[전일 보고 / 템플릿 선택 (선택)]
  ↓
[보고 내용 입력]
  ↓
[제출]
  ↓
[AI 키워드 자동 추출]
  ↓
[1일 보고 완료 화면]
  ↓
[공유 미리보기 (선택)]
  ↓
[카카오톡 공유 버튼 클릭 (Mock)]
```

---

## 7. UX/UI 설계 원칙

### 7.1 Mobile-first

- 모든 화면은 모바일 세로 스크롤 기준으로 설계한다.
- 한 화면 = 하나의 목적을 가진다.

---

### 7.2 Navigation 정책  
### **행동 중심 하단 CTA + 최소 헤더**

#### GNB 정책

- MVP 단계에서는 **GNB(Global Navigation Bar)를 제공하지 않는다.**
- 사용자는 단일 목적 플로우(작성 → 제출 → 공유)를 따른다.

#### 상단 영역 (최소 헤더)

- 현재 화면 타이틀
- 뒤로가기 버튼
- 추가 메뉴 및 탐색 요소는 제공하지 않는다.

#### 하단 영역 (행동 중심 CTA)

- 주요 행동 버튼을 하단에 고정 배치한다.
  - 예: `임시 저장`, `제출`, `카카오톡으로 공유`
- 탐색보다 **행동(Action)** 중심 UI를 유지한다.

---

## 8. 화면 구성 범위

### 페이지(Page) – 총 4개

1. 홈 – 오늘의 보고
2. 1일 보고 작성
3. 1일 보고 제출 완료
4. 공유 미리보기

### 공통 UI 컴포넌트 (Overlay)

- 템플릿 선택 바텀시트
- 이전 보고 선택 바텀시트
- 공유 안내 팝업 / 토스트

---

## 9. 라우팅 구조 (Next.js 기준)

```
/
├─ /                       홈
├─ /reports/daily/new      1일 보고 작성
├─ /reports/daily/complete 1일 보고 제출 완료

```

## 12. 제외 범위 (Explicit Out)

- 로그인 / 회원 관리
- 관리자 페이지
- 실제 AI API 연동
- 카카오 SDK / API Key / CDN 연동
- 알림, 설정, 탐색 중심 기능

---

## 13. 요약

본 MVP는 다음 원칙을 따른다.

- 페이지 기준 **4개 화면**
- GNB 없는 단일 행동 플로우
- 행동 중심 하단 CTA
- 최소 헤더 제공
- 실제 연동 없이도 UX 검증 가능
